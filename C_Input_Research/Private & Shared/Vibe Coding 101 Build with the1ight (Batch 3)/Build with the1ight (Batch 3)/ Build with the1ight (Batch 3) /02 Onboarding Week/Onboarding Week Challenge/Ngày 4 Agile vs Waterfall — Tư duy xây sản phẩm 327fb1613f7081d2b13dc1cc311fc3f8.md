### Mục tiêu

Hôm nay bạn sẽ tìm hiểu 2 cách tiếp cận xây dựng sản phẩm: *Waterfall* (làm tuần tự, xong hết mới ship) và *Agile* (làm nhỏ, ship nhanh, học sớm, sửa liên tục).

Đây là mental model quan trọng nhất cho toàn bộ khoá học. Khi bạn vibe code ở các buổi sau, bạn sẽ tự nhiên làm Agile mà không cần gọi tên nó — nhưng hiểu nó sẽ giúp bạn biết tại sao mình làm vậy.
